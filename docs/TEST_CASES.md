# 简历分析平台测试用例文档

## 1. 测试目标

验证项目是否满足面试测试要求：PDF 简历上传与解析、AI 信息提取、JD 匹配评分、候选人管理、前端体验与工程质量。

## 2. 测试环境

- 前端地址：`http://localhost:3000`
- 后端地址：`http://localhost:8000`
- 健康检查：`http://localhost:8000/api/health`
- 浏览器：Chrome 最新稳定版
- 测试数据：
  - 至少 5 份 PDF 简历。
  - 至少 1 份多页 PDF 简历。
  - 至少 1 份非 PDF 文件，例如 `.txt` 或 `.docx`。
  - 至少 1 份包含姓名、电话、邮箱、城市、教育、工作经历、技能、项目经历的完整简历。

## 3. 测试前置条件

1. 后端已启动：

```bash
source .venv/bin/activate
cd apps/api
uvicorn app.main:app --reload --port 8000
```

2. 前端已启动：

```bash
npm --workspace apps/web run dev
```

3. 后端健康检查通过：

```bash
curl http://localhost:8000/api/health
```

预期返回：

```json
{"status":"ok"}
```

## 10. API 验证用例

### API-001 健康检查

```bash
curl http://localhost:8000/api/health
```

预期：

```json
{"status":"ok"}
```

### API-002 非 PDF 上传被拒绝

```bash
curl -i -X POST http://localhost:8000/api/resumes/upload \
  -F files=@README.md
```

预期：

- HTTP 状态码为 `400`。
- 返回信息说明文件不是 PDF。

### API-003 CORS 预检

```bash
curl -i -X OPTIONS 'http://localhost:8000/api/candidates?sort=created_desc&page_size=24' \
  -H 'Origin: http://192.168.0.101:3000' \
  -H 'Access-Control-Request-Method: GET' \
  -H 'Access-Control-Request-Headers: content-type'
```

预期响应头包含：

```text
access-control-allow-origin: http://192.168.0.101:3000
```

## 11. 验收标准

项目可视为通过核心验收，当满足以下条件：

- 能一次上传至少 5 份 PDF 并完成解析。
- 至少 1 份多页 PDF 能成功提取文本。
- 至少 1 名候选人完成结构化信息提取。
- 至少 1 名候选人完成 JD 匹配评分。
- 候选人能搜索、筛选、排序、查看详情、修改状态。
- 评分结果有图表展示。
- 前端在 1280px 桌面宽度下无明显布局问题。
- `npm run lint`、`npm --workspace apps/web run build`、`npm run test` 均通过。
